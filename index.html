Exploring Markdown

What is Markdown?

Markdown, a play on words of the term "markup", is a lightweight markup language that was designed for readability. It was created and released in 2004 by Mark Gruber. According to Joshua Kehn (2014), "The syntax was inspired by plain text email and readability while allowing simple conversion to html for web publishing." Markdown is an easy way to style text on the web, controlling document display, the formatting of words, addition of images, and the creation of lists.

Benefits of Markdown

According to Heath (2018), some of the benefits of Markdown for Technical Writing includes:

  • Markdown provides semantic meaning for content in a relatively simple way
  • You can write rich formatted content quickly, compared to writing with HTML tags
  • You can read Markdown language and syntax easily in plain text before it is rendered by HTML
  • It doesn't interrupt your workflow with the need to click buttons
  • It's platform-agnostic so your content is not tied to the format of your editor
  • It's lightweight and doesn't require you to learn a lot in order to get started on a project

What's Your Favorite Flavor?

Since its initial release, Markdown has been developed into a number of variations to accommodate different use cases. These different versions are known as "flavors" and offer differing syntax. These Markdown flavors can be implemented within various programming languages and software applications. For example (Kehn, 2014):

  • GitHub deals often with code chunks, so they support fenced code blocks with optional syntax highlighting, tables, and strikeouts.
  • Reddit supports superscript and strikeouts.
  • StackOverflow supports url-autolinking, different syntax highlighting hints, tagging, and an even more limited subset for commenting.

Want to view a comprehensive list of Markdown Flavors? Check out: https://github.com/commonmark/commonmark-spec/wiki/markdown-flavors
